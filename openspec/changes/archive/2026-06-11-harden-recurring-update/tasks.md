- [x] 1.3 In the occurrence-lookup branch (~175-180), select the event whose `occurrenceDate` matches the supplied value within a **~1 ms tolerance (the serialization granularity, not a coarse 1 s window)** **and** whose id matches, instead of `events.first` — e.g.
  `events.first { abs($0.occurrenceDate.timeIntervalSince(occ)) < 0.001 && ($0.eventIdentifier == eventId || $0.calendarItemExternalIdentifier == eventId) }`.
  The 1 ms epsilon absorbs the millisecond quantization of the re-serialized value while staying far below any real occurrence spacing (a 1 s window could alias near-adjacent occurrences of a sub-daily synced series). Add a brief comment that all-day occurrences match on the same midnight instant.
  `EKEvent.occurrenceDate` is a `null_unspecified Date!` (IUO), reliably non-nil for store-predicate results (they always carry a `startDate`); bind it defensively (`guard let od = $0.occurrenceDate`) rather than force-unwrapping, and have `eventToInfo` map over it (`event.occurrenceDate.map(formatISO8601)`) so a nil slot serializes as null instead of trapping. Match `occurrenceDate`, **not** `startDate` (`get_events` emits `occurrenceDate`; a moved occurrence's `startDate` diverges). See design "Match on `occurrenceDate`, with tolerance".
- [x] 1.4 In the occurrence-lookup branch, when no event matches the id + `occurrenceDate` test, decide whether the target series is recurring before erroring, three ways: (a) the window results matched by **either** id form — `events.first { $0.eventIdentifier == eventId || $0.calendarItemExternalIdentifier == eventId }?.hasRecurrenceRules == true`; (b) `store.event(withIdentifier: eventId)?.hasRecurrenceRules == true` for the canonical id; (c) `store.calendarItems(withExternalIdentifier: eventId).contains { ($0 as? EKEvent)?.hasRecurrenceRules == true }` for the external-id / moved-outside-window case (which `store.event(withIdentifier:)` does not resolve). If recurring, throw `occurrenceNotFound` (descriptive, points at `get_events.occurrenceDate`); otherwise keep `eventNotFound`. Branches (a) and (c) honor callers who passed `calendarItemExternalIdentifier` — (a) when the occurrence is still in the window, (c) when it has moved outside it.
